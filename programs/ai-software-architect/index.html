Get the Course arrow_forward
Week 01

Environment and system map

Read the Screeps world, identify available components, and map the signals your colony needs to understand.

01Screeps Environment and System Map

Set up Screeps, connect the development workflow, and read the colony as an environment with resources, components, actions, state, threats, and feedback.

Live: Create the Screeps account or local environment, join the cohort server, create the first room, place the spawn, and map the sources, controller, creeps, memory, terrain, and available actions.Homework: Capture a room screenshot and label the components, constraints, signals, and decisions the colony needs to understand.
02Colony State and Resource Signals

Model the state and signals the colony needs to make better decisions across ticks.

Live: Read room, creep, source, spawn, and controller data, then decide which values should drive harvesting, upgrading, and spawning behavior.Homework: Add three meaningful state signals to the Screeps bot and explain how each one changes a colony decision.
Week 02

Components and decisions

Turn game mechanics into reusable behaviors and teach the colony to make choices when conditions change.

03Components and Repeatable Creep Behavior

Turn game actions into reusable behavior components that make the colony easier to reason about and extend.

Live: Refactor harvesting, transferring, and upgrading actions into named behavior components and use Codex to pressure-test the design.Homework: Create two Screeps behavior components and write notes describing their inputs, outputs, assumptions, and failure cases.
04Logic, Branches, and Room Decisions

Design decision rules that connect colony state to actions under changing resource and room conditions.

Live: Build decision flows for empty creep storage, full creep storage, available energy, spawn capacity, controller upgrades, and immediate room priorities.Homework: Add one Screeps decision rule and document the signal it reads, the action it triggers, and what can go wrong.
Week 03

Git and debugging

Protect strategy experiments, investigate failures, and recover working versions when a live system breaks.

05Git Checkpoints for a Living Bot

Use Git to protect system experiments, compare strategy changes, and recover from broken colony behavior.

Live: Initialize or update the Screeps bot repo, make clean commits, review diffs, and connect version control to design decisions.Homework: Push the Screeps repo and write a README section explaining the current colony architecture.
06Debugging a Running Colony

Diagnose system behavior by inspecting errors, logs, stuck creeps, idle spawns, broken memory, and unclear AI suggestions before changing code.

Live: Diagnose real Screeps failures and practice turning room observations into precise Codex prompts.Homework: Open one repo issue describing a Screeps bug, the evidence, and the next fix to try.
Week 04

APIs, Memory, and roles

Connect game objects, persistent memory, and role-based design to how software systems communicate, remember, and divide work.

07APIs, Game Objects, and Interfaces

Read Screeps game objects, method calls, JSON-like data, and the interface between your code and the game world.

Live: Inspect creep, room, source, controller, and spawn objects, then model how external APIs use similar request and response patterns.Homework: Document one Screeps object or method as if it were an API: what it receives, returns, and changes.
08Memory, Data Flow, and Roles

Organize data so the colony can track roles, tasks, energy needs, room priorities, and strategy over time.

Live: Design Memory structures and connect harvester, upgrader, builder, and defender roles to the behavior components and rules already built.Homework: Add or revise one Screeps Memory structure and explain why it helps the colony scale.
Week 05

Automation and Codex

Use automation and AI support to improve the colony while staying responsible for the decisions your system makes.

09Automation Loops and Scaling

Design automation loops that use triggers, rules, and repeated actions to keep the colony moving without manual intervention.

Live: Build spawn automation, role balancing, repair/build priorities, and fallback behavior while discussing where automation helps and fails.Homework: Add one Screeps automation rule and document its trigger, expected effect, and failure mode.
10Codex as a Screeps Build Partner

Use Codex to pressure-test architecture, plan Screeps features, explain errors, and review work while keeping builder judgment first.

Live: Run a guided Codex workflow: describe a colony improvement, inspect the generated code, test in Screeps, and commit only work you can explain.Homework: Use Codex for one Screeps improvement and write what changed, what was accepted, and what was rejected.
Week 06

Tournament week

Tune a battle branch, play capture-the-flag, and analyze how code-driven strategy performs against another colony.

11Tournament Week: Battle Branch and CTF Prep

Turn the colony architecture into a battle-ready branch for AutoNateAI capture-the-flag rules.

Live: Review the colony mechanics, identify attack and defense priorities, tune the tournament branch, and decide what the bot should protect, chase, ignore, or optimize.Homework: Finish the tournament branch and write a short strategy note explaining how your colony plans to win.
12Tournament Week: Capture-the-Flag Battles

Battle colonies head-to-head in AutoNateAI capture-the-flag, explain the architecture, review tradeoffs, and identify the next version for league-style improvement.

Live: Participants submit tournament branches, run Screeps capture-the-flag matches, watch code-driven colony strategies collide, and analyze how mechanics, state, automation, and architecture shaped the outcome.Homework: Submit final repo link, Screeps match screenshots or clips, architecture notes, tournament reflection, and next-build plan.
