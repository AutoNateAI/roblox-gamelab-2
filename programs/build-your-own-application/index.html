Inside the 3 weeks

01

Session 1: Capstone Proposal

Choose and scope an original capstone system (market trading, defense against scripted NPC invaders, automated expansion, or a non-Screeps app).

Live: Each student pitches their capstone idea and gets feedback before writing a line of code.Homework: Finalize your one-page architecture design doc.
02

Session 2: Architecture Design Doc

Write the architecture plan before touching the AI agent: modules, data flow, state, edge cases.

Live: Peer-review each other's design docs before any building begins.Homework: Complete your architecture design doc if unfinished.
03

Session 3: Build Sprint 1

Implement the capstone's core system with AI assistance, following the design doc.

Live: Working build session; instructor circulates for architecture-level guidance only, not code-level help.Homework: Push at least 3 commits toward the capstone build.
04

Session 4: Build Sprint 2

Iterate, debug, and optimize the capstone system toward a demoable state.

Live: Second build session focused on debugging and polish, applying Program 5 skills.Homework: Have the capstone in a demoable state before next session.
05

Session 5: Peer Playtesting and Review

Give and receive structured peer review, mirroring a real studio code review.

Live: Students test and review each other's capstone systems and leave written feedback.Homework: Incorporate at least one piece of peer feedback before showcase.
06

Session 6: Capstone Showcase - AI Software Architect Certificate

Present the working system and explain the architecture decisions behind it.

Live: Final showcase: each student presents their original system live and fields questions from the group.Homework: Submit final capstone and write-up to the portfolio folder.
