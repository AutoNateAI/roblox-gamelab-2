Debugging Systems

Real debugging under real constraints — Screeps enforces a hard per-tick CPU budget, which forces students to profile, optimize, and handle edge cases (creep death, spawn queue overflow, a scripted NPC invader attacking the room) instead of just adding more code. All threats are automated/scripted opponents on the private server, not other students.

Inside the 3 weeks

01

Session 1: Reading Errors Like a Developer

Learn to read Screeps console errors and visual overlays; set up VS Code for local development.

Live: Deliberately break a working script, then use console errors and overlays to locate the fault.Homework: Screenshot one console error from your own bot and bring it next session.
02

Session 2: CPU Budgeting as a Forcing Function

Understand Screeps' hard per-tick CPU budget and why it forces efficient code.

Live: Profile the bot's CPU usage; identify the most expensive function and optimize it.Homework: Profile your bot and note its average CPU usage per tick.
03

Session 3: Catching Regressions in Diffs

Use diff-checking to catch regressions introduced by AI suggestions before they go live.

Live: Ask the AI for an 'optimization'; diff it against the working version and hunt for a hidden regression.Homework: Practice writing one accept/reject note on an AI diff before next session.
04

Session 4: Defensive Coding for Edge Cases

Handle edge cases defensively: creep death, spawn queue overflow, a scripted NPC invader entering the room.

Live: Simulate a creep dying mid-task and a spawn queue overflow; spawn a scripted NPC invader bot into the room and patch defenses against it.Homework: List 3 edge cases your bot doesn't currently handle.
05

Session 5: Writing Bug Reports the AI Can Act On

Practice directing an AI agent to fix a bug using a clear written description instead of pasted stack traces.

Live: Trade bug reports with a partner and have each person's AI agent fix the other's issue from the written description alone.Homework: Write a clean bug report (no raw error paste) for a real issue in your bot.
06

Session 6: Project Checkpoint - Lead Developer Badge

Confirm the bot survives a stress scenario without crashing or exceeding its CPU budget.

Live: Spawn a scripted NPC invader bot (or a resource crisis) against the bot; verify it survives within its CPU limit.Homework: Record your bot's CPU graph before and after this program for your portfolio.
