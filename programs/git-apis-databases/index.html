Inside the 3 weeks

01

Session 1: Git for a Living Codebase

Initialize Git for the bot codebase; learn commit hygiene and branching for experiments.

Live: Turn the existing bot folder into a Git repo; make a branch to test a risky change safely.Homework: Make 3 small commits with meaningful messages before next session.
02

Session 2: Deploying via the Screeps API

Set up the Screeps CLI/API deploy pipeline (e.g. screeps-api/grunt-screeps).

Live: Configure and run a CLI deploy; push a one-line change and confirm it takes effect live.Homework: Successfully deploy a small change to your live bot via CLI.
03

Session 3: Memory as a Database

Treat Memory as a lightweight database; design a clean schema for roles, room state, and task queues.

Live: Redesign the bot's Memory schema as a small data model; implement it with AI assistance.Homework: Redesign your Memory schema on paper before touching code.
04

Session 4: External Stats Dashboard

Pull live Screeps stats via the API into an external dashboard (tie-in to Airtable or a webhook).

Live: Build a simple external tracker pulling energy/creep-count stats from the Screeps API.Homework: Add one new stat to your external dashboard.
05

Session 5: Reviewing Before Deploying

Use a diff tool to review AI-suggested changes critically before deploying to a live bot.

Live: Ask the AI agent for a refactor; review the diff line-by-line before approving the deploy.Homework: Review an AI-suggested diff and write one sentence on whether to accept it and why.
06

Session 6: Project Checkpoint - Developer Badge

Confirm the full pipeline works end-to-end: Git, CLI deploy, live bot, external stats.

Live: Demo the full pipeline live: commit, deploy, and show the dashboard updating in real time.Homework: Share your Git repo link and dashboard screenshot in the portfolio folder.
