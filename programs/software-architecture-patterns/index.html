Inside the 3 weeks

01

Session 1: Formalizing State Machines

Formalize creep behavior as an explicit state machine (harvesting to returning to idle) instead of implicit if/else logic.

Live: Refactor existing creep logic into an explicit state machine with AI assistance; compare readability before/after.Homework: Diagram your creeps' current states and transitions.
02

Session 2: The Task Queue Pattern

Introduce the task-queue pattern: a central list of jobs that creeps pull from instead of hardcoded roles.

Live: Implement a basic task queue and have idle creeps pull the next available job.Homework: List 3 tasks your bot currently hardcodes that could instead be queued.
03

Session 3: Manager/Worker Pattern

Introduce manager/worker architecture: spawn manager, defense manager, expansion manager as separate concerns.

Live: Split monolithic logic into a spawn manager and a defense manager module with AI assistance.Homework: Sketch your manager modules and what each one owns.
04

Session 4: Scaling to Multiple Rooms

Identify what architecture changes are needed to manage more than one room (shared memory, room roles).

Live: Claim a second room and adapt the manager architecture to coordinate across both rooms.Homework: Write the requirements for claiming a second room, in plain English.
05

Session 5: Predict Before You Prompt

Practice predicting architecture changes before writing the prompt that implements them.

Live: Given a new requirement ('add remote mining'), students write their predicted architecture change, then compare it to what the AI actually builds.Homework: Predict, in writing, what modules a 'remote mining' feature would touch before next session's build.
06

Session 6: Project Checkpoint - Studio Member Badge

Confirm a working multi-room bot with modular manager architecture.

Live: Demo the multi-room bot; present the manager/worker diagram and explain each module's responsibility.Homework: Submit your manager/worker architecture diagram to the portfolio folder.
