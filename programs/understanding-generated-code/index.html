Inside the 3 weeks

01

Session 1: The Tick Loop, Explained

Introduce the Screeps tick loop and how it differs from a normal program's execution model.

Live: Walk through main.js from Program 1's bot together, tracing exactly what runs each tick.Homework: Annotate your own main.js loop line-by-line in plain English.
02

Session 2: Memory as Persistent State

Understand why Screeps needs persistent Memory and how it's structured/serialized.

Live: Inspect Memory in the Screeps console; deliberately clear it and observe what breaks.Homework: Diagram the current Memory schema of your bot (roles, states, targets).
03

Session 3: Finding the Module Boundaries

Identify the module boundaries the AI created (role files, utility files, dispatch logic).

Live: Students trace a single creep's full lifecycle across every file it touches.Homework: List every function in your codebase and what triggers it.
04

Session 4: Naming What You See

Practice naming architecture patterns out loud, not just recognizing them silently.

Live: Pair up and explain your bot's architecture to a partner using only whiteboard/paper, no code.Homework: Name three design decisions the AI made that you wouldn't have thought of.
05

Session 5: Rebuild It Yourself

Rebuild one module without AI assistance to test true understanding.

Live: Students write the harvester role from scratch, unassisted, referencing only their own architecture diagram.Homework: Finish the manual rewrite of the harvester role if not done in session.
06

Session 6: Project Checkpoint - Builder Badge

Compare the manual rewrite to the AI's original version and articulate the differences.

Live: Present your architecture breakdown diagram and manual rewrite to the group; discuss what surprised you.Homework: Submit your architecture diagram to the portfolio folder.
