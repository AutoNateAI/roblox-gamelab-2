Get the Course arrow_forward
Week 01

Environment and system map

Read the Screeps world, identify available components, and map the signals your colony needs to understand.

01Screeps Environment and System Map

Read Screeps as a programmable environment: rooms, creeps, spawns, sources, controllers, terrain, game ticks, state, and available actions.

Live: Set up the workspace, inspect the Screeps API surface, map the first room, and identify the signals a colony needs before it can make useful decisions.Homework: Create an environment map that labels components, constraints, signals, and decisions the colony needs to understand.
02Component Architecture and Role Design

Turn Screeps objects into a colony architecture with roles, behaviors, state, and priority rules.

Live: Design harvester, upgrader, builder, and logistics behaviors; connect them to Memory; and decide how the colony should route work each tick.Homework: Implement or revise two role components and document their inputs, outputs, assumptions, and failure cases.
Week 02

Components and decisions

Turn game mechanics into reusable behaviors and teach the colony to make choices when conditions change.

03Decisions, State, and Colony Automation

Build decision loops that use state, resource signals, spawn capacity, and room priorities to keep the colony moving.

Live: Create spawn automation, role balancing, task switching, and fallback behavior for stuck or idle creeps.Homework: Add one automation loop and document its trigger, expected effect, and failure mode.
04APIs, Interfaces, and Debugging

Use the Screeps API deliberately by reading object contracts, method calls, return values, errors, and live room behavior.

Live: Inspect Game, Room, Creep, StructureSpawn, Source, Controller, Store, and Memory usage; debug broken behavior from logs and visible colony state.Homework: Write an API note for one Screeps object or method and use it to fix or improve one colony behavior.
Week 03

Codex-assisted colony builds

Use Codex with Screeps API context to build project-grade colony subsystems while keeping the architecture explainable.

05AI Build Partner and Feature Planning

Use Codex as an architecture and implementation partner without surrendering judgment.

Live: Choose a feature path, prompt Codex with API-aware context, inspect generated code, test in Screeps, review diffs, and commit only explainable changes.Homework: Use Codex to implement one approved improvement and write what changed, what was accepted, and what was rejected.
06Colony Expansion Projects

Build a project-grade colony subsystem that deepens architecture, API fluency, and tournament readiness.

Live: Select and build one colony project: smarter spawn planner, role scheduler, pathing/cost-matrix helper, repair-and-build planner, tower/defense controller, or energy logistics dashboard.Homework: Finish the project branch, update the README with architecture notes, and identify how the subsystem changes tournament strategy.
Week 04

Tournament week

Tune a battle branch, play capture-the-flag, and analyze how code-driven strategy performs against another colony.

07Tournament Week: Battle Branch and CTF Prep

Turn the colony architecture into a battle-ready branch for AutoNateAI capture-the-flag rules.

Live: Review mechanics, identify attack and defense priorities, tune the tournament branch, and decide what the bot should protect, chase, ignore, or optimize.Homework: Finish the tournament branch and write a short strategy note explaining how your colony plans to win.
08Tournament Week: Capture-the-Flag Battles

Battle colonies head-to-head in AutoNateAI capture-the-flag, explain the architecture, review tradeoffs, and identify the next version for league-style improvement.

Live: Participants submit tournament branches, run Screeps capture-the-flag matches, watch code-driven colony strategies collide, and analyze how mechanics, state, automation, and architecture shaped the outcome.Homework: Submit final repo link, Screeps match screenshots or clips, architecture notes, tournament reflection, and next-build plan.
